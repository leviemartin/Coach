End-to-End Flow: The Connected Pipeline

How data flows from daily input through coaching to next week's plan. One pipeline, no gaps.

DAILY Mon — Sun, throughout the day

Daily Log /log

Morning / Anytime:
Energy level (1-5 tap)
Pain check (0-3 + area if > 0)
See today's suggested session + week overview
Start Session → opens /session
After Gym:
Session summary auto-populated (expandable)
Core work checkbox
Tagged notes (injury / training / life / sleep)
Evening:
Mobility work checkbox
No food after 20:00 checkbox
Hydration logged checkbox
Lights Out button (stamps time)
Sleep disruption tag (if applicable)

Session Tracker /session

Launched from Daily Log:
Can pick any unfinished session for the week
Warns if session breaks sequencing rules
Log sets, reps, weight (as today)
Log cardio rounds/duration (as today)
On Complete:
Writes summary → daily_logs.session_summary
Links session → daily_logs.session_log_id
Sets workout_completed = true
Updates plan_item status → "completed"
Updates plan_item assigned_date → actual date
Checks ceiling progression
Returns to Daily Log
▼ 7 days of data accumulate in SQLite ▼
SUNDAY CHECKIN Sunday 20:00+, after ruck + meals done
STEP 1

Weekly Review

Auto-assembled from daily logs:
Sessions completed (with summaries)
Compliance dashboard
Pain/energy trends (7-day)
Sleep pattern + disruptions
Garmin data overlay
All tagged notes by category

You review, can annotate
STEP 2

Subjective Inputs

Only what daily logs can't capture:
Perceived readiness (1-5)
Plan satisfaction (1-5)
Week reflection (free text)
Next week conflicts
Questions for coaches

Minimal — most data already collected
STEP 3

Triage Agent

AI scans everything and asks:
Clarifying questions (3-5)
Flags missing data
Highlights contradictions
Prompts for backfill

You answer → data is complete
Then: "Submit to coaches"
STEP 4

Coach Synthesis

7 specialists run (streaming):
Each sees ALL data (structured)
Daily notes filtered by domain
Session actuals vs prescribed
Pain/energy daily patterns

Head Coach synthesizes:
Week analysis
Draft plan (sessions + sequencing rules)
STEP 5

Head Coach Chat

Open dialogue:
Ask "why" about any decision
Request swaps or adjustments
Challenge reasoning
Coach explains trade-offs

When satisfied:
"Lock in plan"
Plan writes to DB
Sessions appear on daily log
Week archived
NEW WEEK Monday morning, plan is live
Daily log shows Monday's suggested session
Week overview shows all sessions with sequencing guidance
Swapping a session shows constraint warnings if applicable
Cycle repeats

The Single Pipeline

Daily Log + Session Tracker
  ──write──▸ SQLite (daily_logs + daily_notes + session_logs + session_sets)

Checkin Step 1 ──reads──▸ all daily data + Garmin JSON
Checkin Step 3 ──triage answers──▸ appended to checkin context
Checkin Step 4 ──full context──▸ 7 specialists + Head Coach
Checkin Step 5 ──dialogue──▸ Draft plan refined
"Lock In" ──writes──▸ plan_items (with sequencing) + weekly_metrics + weekly_log archive

Plan ──feeds──▸ Daily Log (next week's sessions)

No data entered twice. No gaps. One direction.
